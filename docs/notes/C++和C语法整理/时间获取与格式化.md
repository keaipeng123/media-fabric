# 时间获取与格式化（sysinfo uptime vs time）

对应源码：

- [SipSupService/src/SipRegister.cpp](../../../SipSupService/src/SipRegister.cpp)

这份笔记总结该文件里出现的两类“时间来源”，以及一个常见的“时间格式化”为字符串的写法。

---

## 1. 时间来源 A：系统运行时长（`sysinfo().uptime`）

在 `SipRegister.cpp` 中，用于计算 `regTime`（注册时间戳）的一种方式是读取 Linux 的系统运行时长：

- 头文件：`#include <sys/sysinfo.h>`
- API：`int sysinfo(struct sysinfo *info);`
- 成功返回 `0`，失败返回 `-1`
- `info->uptime` 表示系统自启动以来的运行秒数（单位：秒）

源码中的典型写法（伪代码化）：

```cpp
time_t regTime = 0;
struct sysinfo info;
memset(&info, 0, sizeof(info));

if (sysinfo(&info) == 0) {
    regTime = info.uptime;  // 秒
} else {
    regTime = time(NULL);   // 兜底
}
```

### 1.1 适用场景

- 更适合做“间隔/超时”的判断：例如 `now - last >= expires`。
- 相比直接使用 `time()`，它不依赖“墙上时钟”（wall clock），一般不会因为 NTP 校时、手动改系统时间导致突然跳变。

### 1.2 局限

- 这是“自开机以来的秒数”，机器重启会从 0 重新开始。
- 它不是 UNIX 时间戳（不是从 1970-01-01 开始算），因此不能直接拿去当日志时间/业务时间展示。

---

## 2. 时间来源 B：UNIX 时间戳（`time(NULL)` / `time(0)`）

同一个文件里也使用了标准 C 时间 API：

- `time_t t = time(NULL);` 或 `time_t t = time(0);`
- 返回值：从 Epoch（1970-01-01 00:00:00 UTC）起的秒数（单位：秒）

该文件中主要用于：生成 SIP 响应里的 `Date` 字段字符串。

### 2.1 适用场景

- 需要“真实日期时间”的业务：例如显示、日志、协议头字段（如 Date）等。

### 2.2 风险点

- 它依赖系统时间（wall clock），如果系统时间被校时/回拨/跳变，可能影响基于差值的超时判断。
- 多线程场景下，配合 `localtime()` 需要注意线程安全（见第 3 节）。

---

## 3. 时间格式化：`strftime()` + `localtime()`

`SipRegister.cpp` 中把 `time_t` 转成字符串的写法是：

- `localtime(&t)`：把 `time_t` 转成当地时区的 `struct tm*`
- `strftime(buf, sizeof(buf), fmt, tm_ptr)`：按格式输出到字符串

源码中的格式字符串是：

- `"%y-%m-%d%H:%M:%S"`

对应含义：

- `%y`：两位年份（00~99）
- `%m`：月份（01~12）
- `%d`：日（01~31）
- `%H`：时（00~23）
- `%M`：分（00~59）
- `%S`：秒（00~60）

示例形状：

```cpp
time_t t = time(0);
char bufT[32] = {0};
strftime(bufT, sizeof(bufT), "%y-%m-%d%H:%M:%S", localtime(&t));
```

### 3.1 线程安全建议

- `localtime()` 返回的是指向静态区的指针，多线程并发调用会发生数据竞争。
- 更稳妥的写法是使用 `localtime_r()`：

```cpp
time_t t = time(NULL);
struct tm tm_local;
localtime_r(&t, &tm_local);
strftime(bufT, sizeof(bufT), "%y-%m-%d%H:%M:%S", &tm_local);
```

### 3.2 协议字段的提示（可选）

如果你需要更标准/更可读的格式，通常会：

- 使用四位年份：`%Y`
- 在日期与时间之间加空格：`"%Y-%m-%d %H:%M:%S"`

具体是否需要 GMT/UTC（例如 SIP `Date` 的期望格式）取决于你对接的平台要求以及 PJSIP 对该头的处理方式。

---

## 4. 一句话结论

- 要做“超时/间隔”的判定：优先考虑 **uptime/单调时间**（本文件用 `sysinfo().uptime` 模拟）。
- 要做“展示/协议日期”：用 **`time()` 得到真实时间**，再用 `strftime()` 格式化。
