# PTS 和 DTS 的区别

本文整理了本次会话中关于 PTS 和 DTS 的核心问题与解答，便于后续查阅。

## 1. PTS 和 DTS 分别是什么

- PTS：Presentation Time Stamp，显示时间戳。
- DTS：Decoding Time Stamp，解码时间戳。

它们分别描述的是媒体数据在播放链路中的两个不同时间点：

1. DTS 决定这一帧应该在什么时候送去解码。
2. PTS 决定这一帧应该在什么时候显示出来。

## 2. 两者的核心区别是什么

最核心的区别是顺序语义不同：

1. DTS 对应解码顺序。
2. PTS 对应播放顺序。

可以简单理解为：

1. DTS 是“先处理谁”。
2. PTS 是“先播谁”。

## 3. 为什么会同时存在 PTS 和 DTS

因为视频流中的解码顺序和显示顺序不一定一致。

当码流中存在帧重排序时，播放器需要：

1. 按某个顺序先把参考帧送去解码。
2. 再按正确的时间顺序把画面显示出来。

这时只用一个时间戳就不够了，所以需要：

1. 用 DTS 指示解码顺序。
2. 用 PTS 指示显示顺序。

## 4. 什么情况下 PTS 和 DTS 可能相同

如果视频中只有 I 帧、P 帧，或者没有发生显示重排序，那么：

1. 解码顺序通常和显示顺序一致。
2. 此时 PTS 和 DTS 往往相同。

音频场景里也经常是这样，因为音频通常不存在复杂的帧重排序。

## 5. 什么情况下 PTS 和 DTS 会不同

如果视频中存在 B 帧，那么 PTS 和 DTS 往往不同。

原因是：

1. B 帧通常依赖前后参考帧。
2. 为了让 B 帧可解码，编码器或发送端往往会先发送参考帧。
3. 这样就会导致解码顺序和最终显示顺序不一致。

## 6. 一个典型例子

假设一段视频的显示顺序是：

```text
I1, B2, B3, P4
```

但为了保证 B 帧可以正确解码，编码或传输顺序可能是：

```text
I1, P4, B2, B3
```

这时：

1. DTS 按解码顺序走：I1, P4, B2, B3。
2. PTS 按播放顺序走：I1, B2, B3, P4。

因此，P4 可能先被解码，但不会立刻显示；B2 和 B3 会在适当时刻按 PTS 被显示出来。

## 7. 在播放器里通常怎么使用

在典型播放链路中：

1. 解码器更关注 DTS。
2. 渲染器和音视频同步模块更关注 PTS。

也就是说：

1. 先根据 DTS 把帧按正确顺序送入解码器。
2. 解码完成后，再根据 PTS 决定什么时候把这一帧显示到屏幕上。

## 8. 一句话总结

一句话概括：

1. DTS 是解码时间戳，决定什么时候解码。
2. PTS 是显示时间戳，决定什么时候播放。
3. 没有帧重排序时两者可能相同；有 B 帧时两者通常不同。
